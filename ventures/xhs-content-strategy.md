# 小红书博主内容规划

> 创建日期: 2026-04-01
> 状态: 规划阶段

## 定位与人设

**核心标签**: 马来西亚程序员 · 数据驱动自律 · AI 效率工具
**差异化**: 用工程师思维量化生活——别人靠感觉，你靠数据和系统

---

## 候选方向

### 方向 A: 程序员自律生活 / "工程师式生活管理" ⭐ 推荐起步
- **内容形式**: 图文卡片、数据可视化截图、对比图
- **选题示例**:
  - "我用代码管理自己的生活，坚持了 X 周后的变化"
  - "程序员的一日三餐，精确到每克蛋白质"
  - "我给自己写了一个熔断机制——连续 3 天睡不够自动砍任务"
  - "数据复盘：我的四维度评分系统（产出/健康/心智/习惯）"
  - "海外程序员的 6:30AM 晨跑 routine"
- **素材来源**: Personal-OS 每日日志、周报数据、评分趋势图
- **学到什么**: 内容创作、数据可视化、写作表达、个人品牌
- **目标受众**: 想自律但缺方法论的年轻人、程序员群体

### 方向 B: AI + 个人效率工具 ⭐ 推荐起步
- **内容形式**: 教程图文、录屏演示、模板分享
- **选题示例**:
  - "我用 Claude 搭了一个 AI 周报系统，自动分析我的生活"
  - "程序员的 AI Agent 工作流实战分享"
  - "Prompt Engineering 实战：让 AI 帮你做每日复盘"
  - "AI 时代程序员的效率工具箱（附模板）"
  - "从零搭建你的 Personal-OS：数据驱动自我管理"
- **素材来源**: Personal-OS 项目本身、Claude 使用技巧、自动化脚本
- **学到什么**: prompt engineering、AI 应用开发、产品设计、技术写作
- **目标受众**: 对 AI 工具感兴趣的职场人、技术爱好者
- **变现潜力**: 最高——模板付费、课程、咨询

### 方向 C: 马来西亚程序员理财
- **内容形式**: 数据图表、对比分析、月度复盘
- **选题示例**:
  - "25岁在马来西亚，存款突破 50K 的路线图"
  - "程序员的记账系统：用 YAML 管理每一分钱"
  - "马股 vs 美股，我的真实收益对比"
  - "月薪 XX 在吉隆坡的生活成本全拆解"
  - "海外打工人的存钱策略"
- **素材来源**: finance/portfolio.yaml、每日消费记录
- **学到什么**: 投资分析能力、财务规划、市场研究
- **注意事项**: ⚠️ 绝不荐股，只分享框架和思路；注意合规

### 方向 D: 程序员健身饮食 / Clean Eating
- **内容形式**: 食物实拍、营养数据卡片、训练记录
- **选题示例**:
  - "程序员的 Meal Prep：一周伙食费不到 RM100"
  - "练后黄金窗口期，我的精确营养配比"
  - "RM15 的鸡胸肉能吃多少顿？成本精算"
  - "久坐程序员的力量训练计划"
- **素材来源**: 每日饮食记录、食材单价表、训练日志
- **学到什么**: 运动营养学、食物摄影、内容呈现

## 执行计划

### Phase 1: 测试期（第 1-4 周）
- [ ] 选定方向 A + B 作为起步组合
- [ ] 注册/完善小红书账号，设定头像、简介、标签
- [ ] 产出 20 篇内容（每方向 10 篇），测试互动率
- [ ] 确定最佳发布时间（测试 7:30AM / 12:30PM / 21:00PM）
- [ ] 建立内容素材库（从 daily/ 和 reports/ 提取可视化素材）

### Phase 2: 聚焦期（第 5-8 周）
- [ ] 根据数据选定 1 个主方向 + 1 个辅方向
- [ ] 建立固定栏目（如"周一数据复盘"、"周三 AI 工具"）
- [ ] 开始与同类博主互动，建立圈子
- [ ] 优化内容模板，提升产出效率

### Phase 3: 增长期（第 9-12 周）
- [ ] 尝试视频/Vlog 内容形式
- [ ] 准备 1 个可分享的免费模板（引流用）
- [ ] 评估变现时机：知识付费模板 > 咨询 > 广告

## 内容产出 SOP

1. **素材收集**: 每日日志自动积累（已有 Personal-OS 支撑）
2. **选题规划**: 每周日规划下周 3-4 篇内容主题
3. **制作流程**: 提取数据 → 设计卡片 → 撰写文案 → 发布
4. **复盘迭代**: 每周检查各篇互动数据，优化选题方向

## 核心原则

1. **内容即学习** — 每篇内容都是对自身知识的结构化输出，倒逼深度学习
2. **数据说话** — 用真实数据和系统截图建立信任，而非空洞的鸡汤
3. **长期主义** — 前 3 个月不考虑变现，专注内容质量和风格确立
4. **低成本试错** — 利用已有的 Personal-OS 数据，最小化额外内容生产成本
